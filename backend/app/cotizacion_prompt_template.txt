Genera una cotización comercial completa y profesional en formato PDF, junto con dos correos: uno para enviar al cliente y otro interno para la empresa. La salida debe ser clara, ejecutiva, bien estructurada y lista para uso real. No incluyas explicaciones sobre el proceso de generación, ni observaciones técnicas de IA, ni textos meta. Entrega únicamente el contenido final. Objetivo general Debes construir una propuesta comercial integral basada en la información suministrada del cliente, del proyecto y del alcance. La cotización debe servir para ser enviada directamente al cliente en PDF y debe mantener una estructura corporativa, comercial y técnica suficiente para proyectos de software, inteligencia artificial, automatización, servicios documentales, acompañamiento técnico, integraciones, biometría, OCR, páginas web, servicios legales o proyectos modulares. Estructura obligatoria del PDF de cotización La cotización debe contener, en este orden, las siguientes secciones: 1. Encabezado comercial Debe incluir: • ciudad y fecha • código o referencia de la cotización • nombre del cliente o empresa cliente • nombre del contacto • nombre del proyecto o servicio • asunto o título de la propuesta 2. Introducción Redacta una introducción ejecutiva que explique: • el contexto general del cliente • la necesidad identificada • el propósito de la propuesta • el valor que aportará la solución 3. Descripción del proyecto Describe claramente: • qué se va a desarrollar, implementar o prestar • cuál es el objetivo del proyecto • qué problema resuelve • qué áreas, procesos o usuarios impacta 
4. Alcance del proyecto Detalla qué incluye la cotización. El alcance debe poder adaptarse según el tipo de proyecto y organizarse por: • fases • módulos • hitos • componentes funcionales • servicios incluidos También debe incluir, cuando aplique: • exclusiones • actividades no incluidas • elementos que requerirán cotización adicional 5. Módulos, componentes o servicios incluidos Para cada módulo o servicio, incluye: • nombre del módulo o componente • objetivo • descripción funcional • actividades incluidas • resultado esperado • integraciones asociadas, si aplica 6. Entregables Incluye los entregables concretos del proyecto, según corresponda, por ejemplo: • desarrollo funcional • código fuente • documentación técnica • documentación funcional • manual de usuario • manual técnico • despliegue o publicación • parametrización • configuración • pruebas • acompañamiento • capacitación • actas o documentos finales • archivos editables y PDF, cuando aplique 7. Tecnologías a utilizar Solo si aplica para proyectos tecnológicos. Debe incluir: 
• frontend • backend • base de datos • infraestructura • servicios cloud • APIs • SDKs • herramientas de IA • herramientas de seguridad • componentes de terceros Si no aplica, omite esta sección sin mencionarlo. 8. Tiempo estimado de ejecución Incluye: • duración total • duración por fase o módulo, si aplica • dependencia de aprobaciones o insumos del cliente • aclaración sobre cuándo inicia formalmente el proyecto 9. Garantía y soporte Especifica: • tiempo de garantía • cobertura de la garantía • qué tipo de soporte se ofrece • qué no cubre la garantía • condiciones del soporte posterior, si aplica 10. Requisitos y supuestos Incluye todas las condiciones necesarias para ejecutar el proyecto, por ejemplo: • accesos • usuarios responsables del cliente • entrega de información • infraestructura • licencias • credenciales • contenidos • branding • tiempos de respuesta del cliente • habilitación de ambientes • disponibilidad de terceros 
11. Inversión o costo del proyecto Presenta la inversión de forma clara y ordenada. Debe soportar cualquiera de estos modelos: • valor total único • valor por fases • valor por módulos • valor por hitos • valor por alternativas comerciales • valor por premisas Debes mostrar: • subtotal • impuestos si aplican • total • moneda • notas aclaratorias si el valor no incluye ciertos conceptos 12. Costos operativos no incluidos Cuando aplique, incluye una sección separada para costos recurrentes o externos no incluidos en la cotización, por ejemplo: • hosting • dominio • licencias • servicios de terceros • consumo de APIs • servicios de mensajería • biometría • OCR • mapas • servidores • almacenamiento • mantenimiento recurrente no incluido 13. Forma de pago Debes incluir la modalidad de pago propuesta, por ejemplo: • anticipo y saldo • pagos por hitos • pagos por fases • pagos por módulo • pagos mensuales • pago contra entrega • pago contra aprobación 
Redáctalo de manera clara, comercial y formal. 14. Consideraciones finales Incluye condiciones generales como: • cambios fuera de alcance serán cotizados aparte • tiempos sujetos a aprobaciones del cliente • cuentas de terceros deben estar a nombre del cliente, si aplica • cualquier nuevo requerimiento se tratará como evolución • la propuesta tiene vigencia comercial por un tiempo determinado • aclaraciones sobre IVA o impuestos, si aplica 15. Datos de pago y cierre comercial Incluye: • nombre de la empresa proveedora • NIT o identificación tributaria, si se suministra • banco • tipo de cuenta • número de cuenta • titular • correo de confirmación de pago • nombre del responsable comercial • cargo • correo • teléfono 16. Firma o cierre final Redacta un cierre comercial profesional, cordial y orientado al inicio de la relación o a la continuidad del proceso. Reglas de redacción para la cotización • Usa tono profesional, ejecutivo y comercial. • El texto debe verse listo para copiar en un PDF formal. • No incluyas notas sobre inteligencia artificial. • No incluyas advertencias meta como “este documento fue generado”. • No incluyas referencias, enlaces, fuentes ni citas. • No inventes información crítica que no haya sido suministrada. Si falta un dato clave, usa un placeholder claro entre corchetes. • Si el proyecto es técnico, usa un lenguaje técnico-comercial equilibrado. • Si el proyecto es legal o documental, prioriza claridad contractual, alcance y entregables. • Si el proyecto es modular, organiza el contenido por módulos. • Si existen varias alternativas comerciales, preséntalas como opción 1, opción 2, premisa 1, premisa 2 o similar. • Si hay dependencias externas, la sección de requisitos y supuestos es obligatoria. 
• Si hay costos recurrentes externos, la sección de costos operativos no incluidos es obligatoria. Datos mínimos que debes usar para construir la cotización Debes trabajar con los siguientes datos de entrada cuando sean suministrados: • nombre de la empresa proveedora • nombre del cliente • nombre del contacto • proyecto • ciudad • fecha • código de cotización • objetivo del proyecto • contexto o problema del cliente • alcance • módulos o servicios • entregables • tecnologías • duración • garantía • requisitos • supuestos • inversión • costos no incluidos • forma de pago • datos bancarios • responsable comercial Formato de salida requerido Debes entregar exactamente tres bloques de contenido: Bloque 1: Cotización PDF Debes redactar el contenido completo del documento en formato formal, con todas sus secciones, listo para maquetar en PDF. Bloque 2: Correo al cliente Debes redactar un correo externo, profesional y cordial, para enviar la cotización al cliente. Debe incluir: • asunto del correo • saludo • mensaje breve explicando que se adjunta la propuesta • resumen ejecutivo corto de lo que encontrará en el documento • cierre cordial 
• firma Bloque 3: Correo interno a la empresa Debes redactar un correo interno para dejar trazabilidad del envío o de la generación de la cotización. Debe incluir: • asunto • resumen del cliente • nombre del proyecto • valor cotizado • modalidad de pago • tiempo estimado • estado del proceso • responsable • observaciones internas si fueron suministradas Plantillas de estilo esperadas El correo al cliente debe sonar comercial, claro y confiable. El correo interno debe sonar corporativo, resumido y útil para seguimiento. La cotización debe ser la pieza principal y tener el mayor nivel de detalle. Instrucción final de comportamiento Con la información suministrada, construye directamente: 1. el texto final completo del PDF de cotización, 2. el correo para el cliente, 3. el correo interno para la empresa, sin agregar explicaciones, sin comentarios sobre el proceso, sin listas de chequeo y sin referencias externas. Si falta información, conserva la estructura y usa campos entre corchetes para completar después.
